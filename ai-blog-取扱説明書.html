AI Blog 取扱説明書

AI Blog は、ブラウザで動くシングルファイルのブログ閲覧・編集アプリです。 記事の作成、Markdown編集、タグ絞り込み、パレット管理、JSON import / export、Supabase Cloud同期、LM Studioを使ったAI補助を利用できます。

このHTML版は、人間が読みやすい章立てと、AIが仕様を抽出しやすい構造化セクションを両方含みます。

単体で動作ai-blog.html を開くだけで、ローカル保存型のブログとして使えます。
同期は任意Supabase URL、anon key、共有IDを設定した場合だけ記事と基本設定を同期します。
AI補助も任意LM Studioを有効にしたPCブラウザだけで、文章作成、読者メモ、保存後コメントを生成します。

起動と初期設定

ローカルだけで使う

  1. output/ai-blog/ai-blog.html をブラウザで開きます。
  2. 既存データがある場合はヘッダーの import からJSONを読み込みます。
  3. 新しい記事を書く場合は Articles 見出し右側の + を押します。
  4. 記事、テーマ、表示モード、下書きなどはこのブラウザの localStorage に保存されます。

Cloud同期とAI補助を使う

  1. ヘッダー右側の歯車ボタンを押して設定を開きます。
  2. Cloud同期を使う場合は、Supabase URL、anon key、共有IDを入力します。
  3. AI補助を使う場合は、LM Studio を起動し、LM Studio を使用する をオンにします。
  4. 保存 または 同期 を押します。
注意: Supabaseの共有IDはパスワードではありません。Supabase URL、anon key、共有IDを知っている人は同じデータを読み書きできます。 個人利用・限定共有向けの設計です。

記事一覧と詳細

初期表示では、記事一覧が240から280px前後のコンパクトな統一タイルで折り返し表示されます。

  • 各タイルには日付、タグ、タイトル、画像、本文抜粋が表示されます。
  • 本文中の最初の画像をサムネイルとして使います。
  • 画像がない記事は画像欄を取らず、テキスト主体のタイルとして表示します。
  • タイトルが空の記事は Untitled と表示されます。
  • 画像なし記事ではタイトル領域を固定し、本文抜粋の開始位置を揃えます。
  • Palette が設定された記事は、カード最下部にカラーストリップが表示されます。
  • 記事タイルとパレットカードは、クリックに加えて Enter / Space でも選択できます。

記事を選択すると、一覧を隠して読み物ページ風の詳細表示が開きます。詳細表示の ← Articles、または画面上部のブログタイトルをクリックすると一覧へ戻れます。

作成・編集・削除

新規記事

  1. Articles 見出し右側の + を押します。
  2. Title、URL、Tags、本文を入力します。
  3. 必要に応じて Palette やAI補助を使います。
  4. 保存ボタンを押して保存します。

選択中の通常タグがある場合、新規記事にはそのタグが引き継がれます。

編集と削除

  1. 記事詳細画面の編集ボタン、または e で編集します。
  2. 保存ボタンで保存し、キャンセルボタンまたは Esc で編集を終了します。
  3. 削除する場合は編集画面の削除ボタンを押し、確認ダイアログでOKを押します。

保存するたびに updatedAt が自動記録されます。

URL・Tags欄

  • ▸ URL・Tags / ▾ URL・Tags で展開・折りたたみできます。
  • URLまたはTagsが入力済みの記事では、編集画面を開いた時に自動で展開します。
  • 折りたたみ時は URLあり / Tags N件 / 未設定 の状態バッジを表示します。
  • 折りたたんでも入力値は保持され、保存、下書き保存、AI補助の参照内容には影響しません。

Markdownと画像

本文はMarkdownで記述できます。保存後はレンダリングされたプレビューとして表示され、コードブロックはシンタックスハイライト付きで表示されます。

スプリットプレビュー

通常編集画面では、本文欄右上の ⊡ Split でテキストエリアとプレビューを左右に表示できます。編集を保存またはキャンセルすると、編集のみの状態に戻ります。

画像の挿入

  1. URL欄にページURLを入力します。
  2. URL欄右端の画像取得ボタンを押します。
  3. 表示された画像候補から使いたい画像を選びます。
  4. テキストエリアのカーソル位置に ![](画像URL) が挿入されます。

画像取得は外部ページへのアクセスとCORSプロキシに依存します。取得できない場合は、画像URLを本文へ直接貼り付けてください。

タグ・検索・ピン

検索

  • ヘッダーの検索欄でタイトル、タグ、本文を検索できます。
  • / で検索欄にフォーカスできます。
  • 入力中に記事一覧がリアルタイムで絞り込まれます。

タグ絞り込み

  • Tagsサイドバーのタグ名で絞り込みます。
  • Articles見出し横のドロップダウンでも単一タグを選択できます。
  • サイドバーでは Shift+クリックで複数タグをOR条件として選択できます。

特殊フィルタ

フィルタ動作
すべて全記事を表示します。
ピン留めピン留めした記事のみ表示します。
タグなしタグが設定されていない記事のみ表示します。

記事詳細のピンボタン、または p でピン留めを切り替えられます。ピン留め記事は一覧で優先表示され、JSON exportにも含まれます。

パレット

記事ごとに最大7色の Palette を保存できます。

  • 編集画面の ▸ Palette でパレット欄を展開できます。
  • カラーピッカーまたはHEX入力で色を設定できます。
  • URL欄にページURLがある場合、抽出ボタンでページ由来の色候補を取得できます。
  • Palette が設定された記事は、記事詳細や記事カードにカラーバーが表示されます。

パレットギャラリー

Articles 見出し右側のパレットボタンで、Palette が設定された記事だけを一覧表示できます。カラーブロックをクリックすると、その色を画面いっぱいに表示します。

ギャラリー表示中は、背景色を暗め・基準・明るめの3段階で切り替えられます。背景色の選択は localStorage に保存されます。

AI補助

編集画面の Palette 欄の下に AI補助 パネルがあります。ヘッダーの設定で LM Studio を使用する がオンの場合だけ表示されます。

機能内容
文章作成入力した材料をもとに、日記形式または論考形式の本文案を生成します。
読者目線のメモ改善するとよい点、もう少し知りたい点、疑問に思う点を短い箇条書きで表示します。
保存後コメント記事保存後に短いポジティブコメントを生成し、必要な場合だけ本文末尾へ追記できます。
  • AI補助欄は通常、1行のバーだけで表示されます。
  • ▸ AI補助 / ▾ AI補助 の文字で展開・折りたたみできます。
  • 生成は明示的に 生成 または 文章生成 を押した時だけ実行されます。
  • 生成結果は自動挿入・自動保存されません。
  • スマホ表示ではLM Studio関連UIとAI補助パネルは非表示です。

文章作成

  1. 文章作成 タブを開きます。
  2. 材料欄に出来事、キーワード、主張などを入力します。
  3. 日記形式 または 論考形式 を選びます。
  4. 文章生成 を押します。
  5. 必要な場合だけ 本文末尾へ追加 を押します。

日記形式は500から900字程度、論考形式は700から1100字程度が目安です。材料欄は最大1500字で、800字を超えると注意表示になります。

保存後コメント

LM Studio が有効で、設定の 記事保存後にコメントを生成する がオンの場合、記事保存後に短いコメントを生成します。コメントを保存 を押した場合だけ、本文末尾に comment: 生成コメント の形式で追記して保存します。

LM Studio設定

LM Studio を起動してローカルAPIサーバーを開始しておくと、PCブラウザでAI補助を使えます。

  1. LM Studio の Developer 画面で Start server を有効にします。
  2. 必要なモデルをロードします。
  3. AI Blog の歯車ボタンをクリックします。
  4. LM Studio を使用する にチェックを入れます。
  5. Base URL に http://localhost:1234/v1 を入力します。
  6. Model欄は空欄でも構いません。空欄の場合は /v1/models の先頭モデルを使います。
  7. LM Studio側でAPI token認証を有効にしている場合は、API Token欄に入力します。
  8. 接続テスト でモデル取得と短い生成を確認します。

推奨モデルは google/gemma-4-31b です。google/gemma-4-e2b でも読者コメント生成まで動作確認済みです。

診断コード

診断コード意味
DISABLEDLM Studio設定がオフです。
EMPTY_BODY本文がないため生成を実行しません。
TIMEOUT生成または通信がタイムアウトしました。
EMPTY_RESPONSELM Studioから本文として使える応答が返りません。
JSON_PARSE期待した形式として解釈できません。
FETCH_ERRORブラウザからLM Studioへ接続できません。
HTTP_400 などLM Studio APIがHTTPエラーを返しました。
API Token はこのブラウザの localStorage に保存されます。共有PCでは入力しないでください。

Supabase Cloud同期

設定モーダルの Cloud 同期 に Supabase URL、anon key、共有IDを入力すると、記事と基本設定を複数端末で同期できます。事前に output/ai-blog/supabase-schema.sql を Supabase SQL Editor で実行してください。

同期されるもの

対象Supabaseテーブル内容
記事ai_blog_entries記事本文、タイトル、URL、domain、tags、palette、pinned、savedAt、updatedAt
基本設定ai_blog_settingsブログタイトル、テーマ、表示モード、パレット背景

同期されないもの

LM Studio の Base URL、Model、API Token、MBTI設定、下書き、検索語、選択中記事、パネル幅、生成キャッシュ、診断表示は同期しません。

初回同期

  • Cloud設定を入れて 同期 または 保存 を押すと、ローカル記事とクラウド記事をIDでマージします。
  • 同じ id の記事は updatedAt が新しい方を採用します。
  • 片方にしかない記事は残します。
  • マージ後の状態を localStorage と Supabase の両方へ保存します。
  • 削除済み記事は、同期保存時にクラウド側の不要IDとして削除されます。
複数端末で使う場合は、最初に基準にしたい端末で同期し、その後に他端末で同じ Supabase URL、anon key、共有IDを入力してください。

スマホ表示

スマホ幅では、一覧と詳細が横スライド式になります。記事をタップすると詳細画面が開き、← Articles で一覧へ戻ります。

  • 記事一覧は区切り線型のリスト表示になります。
  • 画像がある記事は右側に固定サイズのサムネイルを表示します。
  • 詳細と編集は1カラムで表示します。
  • 保存、キャンセル、削除ボタンはタップしやすいサイズになります。
  • LM Studio設定ボタンとAI補助パネルは表示しません。

import / export

ヘッダーの import でJSONを読み込み、export で現在の記事データをJSONとして保存できます。exportファイル名は aiblog-export-YYYY-MM-DD.json です。

AI補助結果、LM Studio設定、画面表示だけの保存後コメント履歴、一時キャッシュ、下書きはJSON exportに含まれません。

トラブル対応

Cloud同期で権限エラーが出る

Supabase SQL Editorで最新版の output/ai-blog/supabase-schema.sql をもう一度実行してください。テーブル作成済みでも、anonロールへの権限やRLS policyが不足している場合があります。

Cloud同期後に記事が期待どおり反映されない

同じ共有IDを使っているか確認してください。同じ記事IDでは updatedAt が新しい方を採用します。どの端末が最新か分からない場合は、基準にしたい端末で先に同期してください。

LM Studioへ接続できない

LM Studio の Developer server が起動しているか、Base URL が http://localhost:1234/v1 になっているか確認してください。ブラウザからlocalhostへ直接fetchするため、環境によっては接続が拒否される場合があります。

AI生成が空応答または途中で切れる

Qwen系などの推論系モデルでは reasoning_content だけで出力上限を使うことがあります。Gemma 4系または通常のチャット向けモデルを使うか、材料を短くして再試行してください。

下書きや設定が消えた

ブラウザの localStorage を削除すると、記事、設定、下書きは消えます。重要な記事は定期的にJSON exportしてください。
